# Caption Writer For Codex

Write captions that sound like the brand, not a template.

## Inputs

Read:

- `context/brand-style.md`
- `context/content-calendar.md`
- `context/best-performers.md`
- `context/compliance-rules.md`

## Output Location

Save captions to `outputs/captions/`.

## Caption Format

```text
# Captions - [Batch Name]

## Post [number]: [Topic]
Platform:
Objective:
Hook:
Caption:
CTA:
Hashtags:
Visual Direction:
Compliance Notes:
Status: draft
```

## Frameworks

Choose and label one framework per caption:

- Hook -> Story -> Lesson -> CTA
- Problem -> Agitate -> Solve
- Before -> After -> Bridge
- List or carousel teaser
- Contrarian take
- Question hook
- Proof -> Insight -> CTA

## Rules

- Lead with a specific hook.
- Use the brand point of view.
- Avoid generic motivational filler.
- Include the visual direction field for the creative designer.
- Flag claims that require approval or evidence.
- Keep hashtags platform-appropriate.
- State missing inputs and assumptions at the top of the output.
- Prefer batch mode for monthly production unless the user asks for one post.
- For Instagram, prefer 3 to 10 targeted hashtags over large generic hashtag blocks.
- Do not use this as the primary writer for LinkedIn, Threads, or X if the platform-specific skill is available.
